Agência▾

Adicionar Nova Agência

Endpoint: POST /agencias

Descrição: Adiciona uma nova agência ao sistema.

Validações:

  • Os campos nome, endereco e telefone são obrigatórios no corpo da requisição.
  • O campo telefone deve conter exatamente 11 dígitos numéricos.
Exemplo de corpo da requisição:
{
    "nome": "Agência Sul",
    "endereco": "Avenida Sul, 789",
    "telefone": "83988776655"
}
Exemplo de corpo da resposta:
{
    "id": 1,
    "nome": "Agência Sul",
    "endereco": "Avenida Sul, 789",
    "telefone": "83988776655"
}

Listar Agências

Endpoint: GET /agencias

Descrição: Retorna uma lista de todas as agências cadastradas.

Exemplo de corpo da resposta:
[
    {
        "id": 1,
        "nome": "Agência Central",
        "endereco": "Rua Principal, 123"
    },
    {
        "id": 2,
        "nome": "Agência Norte",
        "endereco": "Avenida Norte, 456"
    }
]

Buscar Agência por ID

Endpoint: GET /agencias/{id}

Descrição: Retorna os detalhes de uma agência específica com base no ID fornecido.

Exemplo de corpo da resposta:
{
    "id": 1,
    "nome": "Agência Central",
    "endereco": "Rua Principal, 123"
}

Atualizar Agência

Endpoint: PUT /agencias/{id}

Descrição: Atualiza os dados de uma agência específica com base no ID fornecido.

Validações:

  • Os campos nome, endereco e telefone são obrigatórios no corpo da requisição.
  • O campo telefone deve conter exatamente 11 dígitos numéricos.
Exemplo de corpo da requisição:
{
    "nome": "Agência Central",
    "endereco": "Rua Principal, 123",
    "telefone": "83998765432"
}
Exemplo de corpo da resposta:
{
    "id": 1,
    "nome": "Agência Central",
    "endereco": "Rua Principal, 123",
    "telefone": "83998765432"
}

Remover Agência

Endpoint: DELETE /agencias/{id}

Descrição: Remove uma agência específica com base no ID fornecido.

Cliente▾

Adicionar Novo Cliente

Endpoint: POST /clientes

Descrição: Adiciona um novo cliente ao sistema.

Validações:

  • Os campos nome, cpf e telefone são obrigatórios no corpo da requisição.
  • O campo cpf deve ser no formato 111.222.333-45.
  • O campo telefone deve conter exatamente 11 dígitos numéricos.
Exemplo de corpo da requisição:
{
    "nome": "Carlos Souza",
    "cpf": "111.222.333-44",
    "telefone": "839"
}

Listar Clientes

Endpoint: GET /clientes

Descrição: Retorna uma lista de todos os clientes cadastrados.

Exemplo de corpo da resposta:
[
    {
        "id": 1,
        "nome": "João Silva",
        "cpf": "123.456.789-00"
    },
    {
        "id": 2,
        "nome": "Maria Oliveira",
        "cpf": "987.654.321-00"
    }
]

Buscar Cliente por ID

Endpoint: GET /clientes/{id}

Descrição: Retorna os detalhes de um cliente específico com base no ID fornecido.

Exemplo de corpo da resposta:
{
    "id": 1,
    "nome": "João Silva",
    "cpf": "123.456.789-00",
    "endereco": "Rua Exemplo, 100"
}

Atualizar Cliente

Endpoint: PUT /clientes/{id}

Descrição: Atualiza as informações de um cliente existente com base no ID fornecido.

Validações:

  • Os campos nome, cpf e telefone são obrigatórios no corpo da requisição.
  • O campo cpf deve ser no formato 111.222.333-45.
  • O campo telefone deve conter exatamente 11 dígitos numéricos.
Exemplo de corpo da requisição:
{
    "nome": "João da Silva",
    "cpf": "123.456.789-00",
    "endereco": "Rua Exemplo Atualizada, 100"
}

Remover Cliente

Endpoint: DELETE /clientes/{id}

Descrição: Remove um cliente existente com base no ID fornecido.

Conta Corrente▾

Buscar Conta Corrente por ID

Endpoint: GET /contas/{id}

Descrição: Retorna os detalhes de uma conta corrente específica com base no ID fornecido.

Exemplo de corpo da resposta:
{
    "id": 1,
    "agenciaId": 1,
    "clienteId": 1,
    "saldo": 1000.00
}

Listar Contas Correntes

Endpoint: GET /contas

Descrição: Retorna uma lista de todas as contas correntes cadastradas.

Exemplo de corpo da resposta:
[
	{
	    "id": 1,
	    "agenciaId": 1,
	    "clienteId": 1,
	    "saldo": 1000.00
	},
	{
	    "id": 2,
	    "agenciaId": 1,
	    "clienteId": 2,
	    "saldo": 2500.00
	}
]

Sacar

Endpoint: POST /contas/{id}/sacar

Descrição: Realiza um saque de uma conta corrente específica.

Validações:

  • O campo valoré obrigatório e deve ser maior que zero.
Exemplo de corpo da requisição:
{
    "valor": 200.00
}

Depositar

Endpoint: POST /contas/{id}/depositar

Descrição: Realiza um depósito em uma conta corrente específica.

Validações:

  • O campo valoré obrigatório e deve ser maior que zero.
Exemplo de corpo da requisição:
{
    "valor": 500.00
}

Transferir

Endpoint: POST /contas/{id}/transferir

Descrição: Realiza uma transferência entre contas correntes.

Validações:

  • O campo valoré obrigatório e deve ser maior que zero.
  • O campo numeroContaDestinoé obrigatório.
Exemplo de corpo da requisição:
{
    "valor": 300.00,
    "numeroContaDestino": 10002,
}

Transação▾

Extrato Geral

Endpoint: GET /transacoes/extrato-geral

Descrição: Retorna o extrato geral de todas as transações.

Exemplo de resposta:
[
    {
        "id": 1,
        "contaId": 1,
        "tipo": "SAQUE",
        "valor": 200.00,
        "data": "2024-08-13T12:34:56"
    },
    {
        "id": 2,
        "contaId": 1,
        "tipo": "DEPOSITO",
        "valor": 500.00,
        "data": "2024-08-12T10:00:00"
    }
]

Extrato Mensal

Endpoint: GET /transacoes/extrato-mensal/{mes}

Descrição: Retorna o extrato das transações de um mês específico.

Exemplo de resposta:
[
    {
        "id": 3,
        "contaId": 2,
        "tipo": "TRANSFERENCIA",
        "valor": 300.00,
        "data": "2024-07-15T09:00:00"
    }
]

Extrato Anual

Endpoint: GET /transacoes/extrato-anual/{ano}

Descrição: Retorna o extrato das transações de um ano específico.

Exemplo de resposta:
[
    {
        "id": 4,
        "contaId": 1,
        "tipo": "SAQUE",
        "valor": 100.00,
        "data": "2024-01-10T11:00:00"
    }
]

Extrato Filtrado

Endpoint: GET /transacoes/extrato-filtrado

Descrição: Retorna o extrato das transações filtradas por tipo e/ou período.

Exemplo de corpo da requisição:
{
    "tipo": "DEPOSITO",
    "dataInicio": "2024-01-01",
    "dataFim": "2024-08-13"
}
